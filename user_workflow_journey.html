User Workflow Journey: Parent Fund Management for Youth Account

Experience Mindset

User: Parent/Guardian
Experience: Managing a youth account to teach financial responsibility, including allocating funds, setting spending limits, and monitoring activity.

  • Awareness & Onboarding
  • Account Management
  • Fund Allocation
  • Spending Control
  • Activity Monitoring
  • Troubleshooting (e.g., insufficient funds)

Scenarios & Variations

Scenario 1: Access Youth Account Dashboard

Context: Parent is logged into the banking web application.
Action: Navigates to the youth account section.
Goal: Quickly view youth account balance and recent activity.

  • Variation A: Dashboard is accessible from main navigation.
  • Variation B: Dashboard is accessible via a personalized homepage widget.

Scenario 2: Allocate Funds to Youth Account

Context: Parent is on the youth account dashboard.
Action: Selects “Add Funds” to transfer money.
Goal: Seamlessly transfer funds from parent to youth account.

  • Variation A: Inline modal for fund transfer.
  • Variation B: Dedicated fund transfer page with step-by-step guidance.

Scenario 3: Set Spending Limit

Context: Parent is managing youth account settings.
Action: Configures a weekly spending limit.
Goal: Control and monitor youth spending.

  • Variation A: Limit set via dashboard quick action.
  • Variation B: Limit set via a dedicated settings page with educational tips.

Scenario 4: View Youth Spending Activity

Context: Parent is on the dashboard.
Action: Selects activity section.
Goal: Review recent transactions and spending patterns.

  • Variation A: Inline expandable transaction list.
  • Variation B: Full-page transaction history with filters.

Scenario 5: Handle Insufficient Balance During Fund Allocation

Context: Parent attempts to transfer funds.
Action: Source account lacks sufficient balance.
Goal: Prevent failed transfers and inform the user.

  • Variation A: Inline error message with actionable suggestions.
  • Variation B: Error modal with links to deposit funds or select another account.

Minimum Viable Experience (MVE)

User Goal

  • To efficiently allocate, control, and monitor funds for their child’s youth account.
  • To teach financial responsibility while maintaining oversight.

Business Goal

  • Increase engagement with youth account products.
  • Build trust with parents through transparency, control, and ease of use.
  • Reduce support costs by preventing errors and confusion.

  2. Youth Account Overview Widget (1.1)
  3. Dedicated Fund Transfer Page (3.0)
  4. Spending Limit Settings Page (4.0)
  5. Full Transaction History Page (5.0)
  6. Error Modal: Insufficient Funds (Er.2)
  7. Email Notification: Transfer Success (Em.1)

Screen Details

ScreenGoalDescriptionDesign ProblemsDesign Opportunities

Edge Cases & Accessibility Considerations

  • Parent has multiple youth accounts: UI must allow easy switching and clear context.
  • Parent with visual impairments: All screens must be screen-reader compatible, with high contrast and keyboard navigation.
  • Network failures: Show clear error/retry options without losing entered data.
  • Mobile/tablet views: Responsive layouts for all screens, touch targets sized for accessibility.
  • Input validation: Prevent invalid amounts, limits, and highlight errors in context.

Summary of Screen Sequences

  • Workflow 1: 1.0 Login & Main Navigation → 2.0 Dashboard → Pu.1 Add Funds → 2.1 Set Limit → 2.2 Activity → Er.1/Pu.2
  • Workflow 2: 1.0 Login & Homepage → 1.1 Overview Widget → 3.0 Fund Transfer → 4.0 Limit Settings → 5.0 History → Er.2/Em.1

Design Principles for Accessibility & Scalability

  • Consistent navigation and action placement across flows.
  • Clear, concise language and contextual help for all actions.
  • Scalable layouts for new features (e.g., multiple youth accounts, additional controls).
  • Accessible color contrast, focus indicators, and ARIA labels.
  • Support for localization and different currencies.
